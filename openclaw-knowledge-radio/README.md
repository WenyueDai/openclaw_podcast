# openclaw-knowledge-radio

A reproducible, Raspberry-Pi-friendly daily knowledge pipeline:

- Collect (RSS) within a configurable lookback window (default: last 48 hours)
- Deduplicate across days using a local `state/seen_ids.json`
- Semantic summarization + podcast script via **OpenRouter** (OpenAI-compatible API)
- Write **Obsidian** Daily note (minimal, link-first)
- Chunked TTS via **edge-tts** and MP3 merge via **ffmpeg**
- Outputs to `output/YYYY-MM-DD/` and Obsidian vault `Daily/YYYY-MM-DD.md`

Designed so **OpenClaw** only needs to run one command daily:
```bash
./.venv/bin/python run_daily.py
```

## Quick start (Raspberry Pi)

1) Clone into your workspace (example):
```bash
cd /home/eva/openclaw_workspace
git clone <YOUR_GITHUB_REPO_URL> openclaw-knowledge-radio
cd openclaw-knowledge-radio
```

2) Set your OpenRouter key (required for LLM script):
```bash
export OPENROUTER_API_KEY="sk-or-..."
```

3) Install environment:
```bash
bash install.sh
```

4) Run once:
```bash
./.venv/bin/python run_daily.py
```

You should get:
- `output/YYYY-MM-DD/podcast_YYYY-MM-DD.mp3`
- `obsidian_vault/Daily/YYYY-MM-DD.md` (path set in config)

## Configuration

Edit `config.yaml`:

- `lookback_hours`: only fetch items published/updated in last N hours (default 48)
- `rss_sources`: list of feeds + tags
- `paths.obsidian_vault`: your Obsidian vault folder
- `podcast.voice`: edge-tts voice
- `llm.model`: OpenRouter model id, e.g. `openai/gpt-oss-120b`

## Notes

- This repo intentionally keeps Obsidian notes minimal: one line per item, always includes source URL.
- Podcast script is generated by LLM, but bounded by `llm.max_output_tokens` to avoid runaway length.
